Detailed Description

The singleton that represents the entire game. What it subclasses, MonoBehaviour, is from the Unity engine; this is a variety of GameObject, which is the first object created when a Unity game boots up. It is this object, and this object only for Rain World, that can actually call Unity routines, or interact with Unity in any way. By its nature, it is also the head of the call stack that leads to the update loop that provides for actual gameplay. The full tree initializations is:

  • RainWorld (singleton initialized by Unity engine)
    • ProcessManager (singleton initialized by RainWorld.Update)
      • RainWorldGame (singleton initialized by ProcessManager.SwitchMainProcess every cycle)
        • OverWorld
          • AbstractRoom (initialized by OverWorld.WorldLoaded)
            • Room
          • World (initialized by OverWorld.WorldLoaded)
        • AbstractCreature (initialized by RainWorldGame.RainWorldGame)
          • Player (initialized by AbstractCreature.Realize when creatureTemplate.TopAncestor().type is CreatureTemplate.Type.Slugcat)
        • RoomCamera (initialized by RainWorldGame.RainWorldGame)

See each class's description for its detailed role in the chain. There is a similar tree for frame-by-frame updates:

  • RainWorld.Update (called by the Unity engine)
    • ProcessManaer.Update
      • RainWorldGame.RawUpdate (every "fixed" frame, RainWorld.evenUpdate bool alternating between updates)
        • RainWorldGame.Update
          • Room.Update
            • UpdateableAndDeletable.Update
            • GraphicsModule.Update
            • PhysicalObject.GraphicsModuleUpdated
        • RainWorldGame.GrafUpdate
          • RoomUpdate.DrawUpdate
            • SpriteLeaser
              • IDrawable.DrawSprites
